# 故事神谕（Story Oracle）—— SillyTavern 扩展

一个可拖动的侧边面板，连接到你自定义的 **OpenAI 兼容** 端点，让你针对*正在进行中*的
剧情向 LLM 提问。问答在它自己独立的窗口、自己独立的历史里进行——**完全不会碰到你的主聊天**。
随时可以清空。

## 安装（手动）

把整个 `story-oracle/` 文件夹复制到你的 SillyTavern 第三方扩展目录，然后刷新浏览器标签页。

- **较新版 ST（按用户分目录的数据结构）：**
  `SillyTavern/data/<你的用户名>/extensions/story-oracle/`
  （默认用户名是 `default-user`）
- **较旧版 ST：**
  `SillyTavern/public/scripts/extensions/third-party/story-oracle/`

刷新后，打开**魔棒菜单**（消息输入框旁边的 🪄 图标）→ **故事神谕**。

## 设置

点击窗口标题栏里的 ⚙ 齿轮。

**直连模式（Direct，默认）** —— 完全自包含：
- **端点 URL**：你的基础 URL（例如 `https://your-proxy.com/v1`）。如果你只填了主机名或
  `/v1` 形式的基础地址，会自动补上 `/chat/completions`。如果想绕过自动补全，直接粘贴
  完整的 `.../chat/completions` URL。
- **API 密钥**、**模型**（例如 `gpt-4o-mini`，或你的代理所提供的任意模型）。
- 以明文形式保存在 ST 的 settings.json 中 —— 对个人/自建实例来说没问题。

**连接配置文件模式（Connection Profile）** —— 选择你已保存的某个 ST 连接配置文件。请求会
通过 ST 自己的后端转发，因此 **CORS 无法拦截它**。如果直连模式抛出网络/CORS 错误，就用这个。

共用选项：温度、最大 token 数、**上下文深度**（往里喂多少条最近的消息；`-1` = 整段聊天，
`0` = 不喂）、是否包含角色卡的开关，以及一段可编辑的系统提示词。

## 用补全预设作为系统提示词来源（挑选 + 忠实组装）

除了直接写系统提示词，你还可以让神谕**沿用某个 Chat Completion 补全预设的结构**。在设置里的
**系统提示词来源（补全预设）** 下拉中选择一个预设即可。

**挑选（Curation）。** 第一次选中某个预设时，会弹出一个挑选清单：

- 列出该预设里**全部**的块（按预设里的真实顺序），以及 `角色卡描述 / 性格 / 场景 / 世界书 /
  用户人设 / 对话历史` 等**上下文插槽**（markers）。其中在预设里**已停用**的块也会列出，标上
  「预设已停用」并默认不勾选——和 ST 的预设管理器一一对应，你也可以临时把它勾上。
- 每一行都按名称 / 角色 / 结构做了**默认勾选**：故事 / 世界 / 人格类与插槽默认保留；可见的思维链、
  字数、禁词、文风、人称、起手（assistant/user 预填）等输出/反模式块默认去掉；变量初始化母块
  （大清洗 / 初始化变量这类，按结构而非名称识别）作为隐形机制默认保留。标着「待确认」的是没法
  自动归类的块，请自行决定。
- **可拖动调整顺序**，勾选框决定去留。底部显示当前保留的块数与字数。**保存并使用** 即生效。

**冻结快照。** 保存后，你的挑选结果会被**冻结**进神谕自己的设置里（按预设名保存）。之后即便你在
ST 里改动了原预设，也**不会**影响神谕这份副本——只有点 [↺ 重新挑选] 才会重新读取并让你重新挑。

**忠实组装。** 发送时，神谕会按你挑选并排好序的块**忠实复刻 ST 主聊天的提示词结构**：

- 各插槽就地填入神谕的上下文（角色卡字段、按位置拆分的世界书[常驻+关键词]、用户人设等）。
- **对话历史插槽**会展开为：先是剧情上下文（最近的主聊天消息），再接神谕自己的问答——都作为带
  角色的真实消息。
- **每个块都保留它自己的角色**（system / assistant / user 不会被压成一条系统消息）。
- 块里的宏照原样保留、发送时实时解析（`getvar/setvar` 等）。`setvar` 会**自愈**——主聊天下一次
  生成时母块会重新写回这些变量，所以不会污染主聊天。

**注意：选用预设时，下方的系统提示词文本框（以及内置的简短提示词）会被完全停用**，不再作为兜底。
要回到自定义文本框，把下拉切回「自定义」即可。

**输出正则与渲染（仅预设模式）。** 选用预设、且打开了「应用剧情正则」时，神谕会像主聊天一样
**渲染**它自己的回复——用 ST 自带的消息渲染逻辑过一遍，这样显示用的 `AI_OUTPUT` 正则、Markdown、
以及正则生成的 HTML/CSS 小组件（例如状态栏美化）都会正常显示，而不是把源码当成纯文本贴出来。
同时，写回神谕**对话历史**的是「提示词阶段」正则处理后的文本（也就是主聊天会回喂给模型的那一份），
这样下一轮不会把自己的思维链 / 状态栏 / 小组件源码再读回去。普通文本框模式与诊断模式下不做这步。
**首次挑选预设时会给出一次提醒**：这些正则是按主聊天的输出格式写的，如果你在神谕里挑选的预设和主聊天
当前启用的预设不一样，正则可能匹配不上神谕的输出、清理 / 美化不会生效。

**机制区块自动隐藏（普通 / 参谋模式）。** 此外，喂给神谕的**剧情记录**里也会剥掉各条消息内嵌的
\<UpdateVariable\> 区块（诊断模式除外）：这些区块是增量不是状态，神谕看不到真实变量基线时，留着它们只会
诱导模型自信地报出离谱数值——要查真实数值请用诊断模式，要让方案贴合数值请用参谋模式（它读的是实时 stat_data）。

 有些预设带有「每条回复末尾必须输出 \<UpdateVariable\>」之类的
输出规则（MVU 卡常见），忠实组装会让神谕也照办——于是回复尾巴上挂出一个对主聊天毫无意义的变量区块。主聊天里
它由 MVU 自己消化隐藏，神谕没有那条管线。从 v1.14.1 起，普通与参谋模式会把回复中的 \<UpdateVariable\>
区块从**显示与神谕历史**中一并剥除（诊断模式不受影响——读这些区块正是它的本职）。不想让神谕收到这条规则的话，
也可以在 [↺ 重新挑选] 里直接把对应的变量更新规则块去掉。

## 说话人格与预设

说话人格（普通 / 泉こなた / 贝阿特丽丝 / 胡蝶忍）会给神谕套一层说话腔调。**选用预设时人格默认关闭**，
以免和预设自带的角色声线冲突；如果你*主动*选了某个人格，它会作为一条系统消息**叠加在预设之上**。
选「普通」＝不叠加任何人格。人格在普通 / 参谋 / 世界书模式下均可叠加：参谋与世界书模式会自动附加一段「职责调整 + 结构保护」——人格只改变说话的语气，方案区块的格式、写入世界书的条目正文都不受人格影响（人格属于神谕，不属于世界书）。诊断模式下人格不生效。

## 诊断模式（Diagnose）

点击标题栏里的 🩺 听诊器图标可切换到**诊断模式**，用于修复 MVU（MagVarUpdate）的状态变量。

在该模式下，神谕会读取本角色卡的 MVU 规则（来自世界书）、当前的实时变量状态（`stat_data`），
以及最新一条 AI 回复里的 `<UpdateVariable>` 区块，然后对照剧情进行核对，最后给出一份保守的
纠正补丁。当回复中包含可用的补丁时，气泡下方会出现一个 **将修复应用到状态** 的按钮——它会
通过 MVU 自己的管线应用补丁，并支持一键**撤销**。

> 提示：「审计整个状态」这类全量审计输出较长，推理型模型还会先消耗一部分 token 思考。诊断模式已自动把
> token 上限抬到至少 4096；如果补丁仍被截断或回复为空，请在设置里继续调大「最大 token 数」，或把问题问得更聚焦（例如只审某一类变量）。

## 世界书模式（Lorebook）

点击标题栏里的 📖 书本图标可切换到**世界书模式**，用来**聊世界书**，也可以**直接修改世界书**。
（与诊断 / 剧情参谋模式互斥，开一个会自动关掉其它。）

进入该模式后，标题下方会出现一条工具栏：

- **世界书下拉**：默认是「当前激活的全部世界书」（角色卡 / 聊天 / 全局所绑定的那些，带 ★ 标记），
  也可以直接选某一本来专门处理。点 ↺ 可刷新列表。
- **同时带上最近剧情对话**：默认关闭——世界书模式聚焦于世界书本身而非当前 RP；需要结合剧情来判断时再勾上。
- **套用我的补全预设（指令叠加其上）**：默认关闭。勾选后，世界书模式会**走你在设置里整理好的补全预设**，把世界书管家指令叠加在预设之上——
  保留预设里的文本块（越狱 / 格式 / 框架；后置越狱块仍排在最后），但跳过角色卡 / 世界书 / 人格等 RP 用的占位块（它们与改书无关）。
  ⚠ 仅在**确实需要越狱**才勾：预设的额外内容会分散模型注意力、可能影响编辑精度。没整理过预设时勾它不报错，会自动退回内置管家提示词。
- **条目选择器**（选定某一本书时出现）：默认发送整本，但你可以展开它、用复选框挑出**只发哪几条**给神谕讨论。
  自带筛选框和几个快捷按钮：「全选 / 全不选」，以及按类型一键选「仅蓝灯（常驻）/ 仅绿灯（关键词触发）/ 仅禁用」——
  前两者都**不含已禁用**条目。挑选只影响**发给 AI 的内容**——应用改动时仍会重新读取整本书，所以即便只发了几条，编辑任何 uid 也照常生效。标题栏会注明「已选 X / 共 N」。

神谕会把选定世界书的条目读进来（每条带 uid、标题、关键词、类型蓝灯/绿灯/禁用、位置与顺序、正文；若用了条目选择器则只读你勾选的那些），于是你可以：

- **聊**：里面都写了些什么、某个设定写在哪一条、条目之间有没有矛盾、帮我想想这个设定还能怎么扩写……（此时只回答，不改动）。
- **改**：让它改写某条内容、补一条新设定、删掉某条、改标题或关键词、把某条设成常驻或禁用……

当你要求改动时，神谕会先用中文说明改了什么，并在末尾附上一个或多个 `<LorebookEdit>` 区块。
这些区块用的是**分隔（delimited）格式**而不是 JSON：元信息按「键: 值」逐行写，长正文放进 `<<<content … content>>>`
这样的「围栏」里**原样书写、无需任何转义**——所以正文里的引号、冒号、换行、`<标签>` 都不会再把解析搞坏（这正是
旧 JSON 格式会偶发失败的地方）。每个区块是一处独立改动，某个写坏了也不连累其它。

四种操作：`create`（新建）、`edit`（整条覆盖，仅用于很短或确实要重写的条目）、`patch`（**修改已有条目的默认方式**——
用「开头几个字 || 结尾几个字」锚定一段范围、只替换那一段，不必重发整条）、`delete`（删除）。

气泡下方随即出现一个 **将改动应用到世界书** 的按钮，并标出本次改动概览（如 `新增 1 · 补丁 1`）。点下后：

- 通过 ST 自己的 `saveWorldInfo` 把改动写回对应世界书，并逐条列出结果（✓ 已应用 / ⤫ 跳过+原因）。
- 支持一键**撤销**（把每一本被改动的世界书还原到改动前的快照）。
- 改动只会落在工具栏里列出的那些世界书上；uid 不存在、`patch` 锚点找不到、或世界书不在范围内的操作都会被**安全跳过并说明原因**，绝不半途写入。
- 写入的正文**原样保存、不做宏替换**（`{{user}}`、人物本名、缩进结构都原封不动），所以条目在世界书编辑器里是人类可读的、并与本书既有风格一致。

> 提示：整本世界书会一次性注入，token 消耗可能较大。条目很多时，建议在下拉里选定**某一本**来处理，而不是「全部激活」。
> 世界书模式默认不走补全预设，用的是它自己的世界书管家提示词；只有勾上「套用我的补全预设」时才会把指令叠加到预设之上（见上）。若你主动选了某个说话人格，它会叠加在管家指令之上——人格只改语气，区块格式与写入条目的正文不受影响。
>
> 新建条目默认会勾上 **非递归（excludeRecursion）** 与 **不触发后续递归（preventRecursion）**（避免关键词级联）；
> 如果某条确实需要参与递归，可让神谕在该条上把对应项显式设为关闭。可编辑字段也包含这两个递归开关。
>
> 万一某个改动区块实在解析不出来（例如回复被截断、围栏没闭合），按钮位置会直接显示原因，而不是悄无声息地消失——
> 这时让神谕「按格式重发一次」即可。

## 剧情参谋模式（Story Advisor）⚠ 实验性

点击标题栏里的 🧭 罗盘图标可切换到**剧情参谋模式**——和神谕一起**构思剧情接下来怎么走**，
并可一键让主聊天**被悄悄引导**向选定的方向。（与诊断 / 世界书模式互斥。）

> ⚠ **实验性功能**：引导效果高度依赖你所用的模型与预设，措辞与行为可能在后续版本调整。

**构思。** 进入该模式后，神谕会通读**整段**对话（无视设置里的上下文深度），帮你找沉睡的伏笔、
未兑现的线索、人物弧光的下一步。如果角色卡使用 MVU 变量框架，参谋还会读取**当前实时变量状态**
（stat_data：好感度、金钱、时间等），并把这些数值当作剧情硬事实——提出的方案不会与数值现状矛盾。讨论收敛后，它会把可落地的走向列成**方案卡片**（来自回复里的
`<StoryPlan>` 区块），每张卡片包含：目标（一句话的结果）、起始迹象、契合点。

**从普通模式无缝接力。** 神谕的侧聊历史在各模式间共享：如果你先在普通模式（带不带人格都行）和神谕聊出了感觉，切到参谋模式时会出现一个【🧭 把刚才的讨论整理成方案】按钮——点一下，参谋就会把刚才的讨论正式化成可采用的方案卡片。（按钮只在「侧聊有内容、且当前没有方案在引导」时出现。）

**引导。** 在卡片上选好**强度**、点【开始引导】，方案就会作为一条幕后指令注入主聊天的提示词
（通过 ST 的 `setExtensionPrompt`，默认插入深度 4、system 角色，可在设置里改「剧情引导注入深度」）。
之后你**正常进行 RP 即可**——主线 AI 会逐步把剧情推向该方向。指令里写明了：自然融入、先铺垫后兑现、
不替 {{user}} 行动、**用户的行动永远优先**（你偏离时它会跟随你，不会强行拉回）。

三档强度（标签即指令的浓缩版，选中后下方会显示一行说明）：

- **只铺垫** —— 只埋伏笔与暗示，暂不让事件正面发生；
- **自然推进** —— 每个场景向目标靠近一小步，时机成熟时自然引发；
- **尽快引爆** —— 在接下来一两个场景内让事件正面发生（仍以已有铺垫为基础）。

**方案条。** 采用后，窗口顶部会出现一条**方案条**（任何模式下都可见，只要方案还在引导）。
**关掉神谕窗口也不会丢**：只要还有方案在引导，方案条会自动搬进一个**独立的悬浮小窗**常驻在 ST 界面上
（可拖动、位置会记住，点 🌙 可随时打开神谕本体）；重新打开神谕时它又会搬回窗口顶部——两处是同一条方案条，
强度、查看注入内容、完成 / 放弃在哪边都能操作。手机上嫌它占地方的话，点 ⌄ 可把悬浮窗**收起成一枚
「🧭 引导中」小标签**（仍可拖动；桌面端悬停可看当前目标），点 ⌃ 再展开；收起状态会被记住。
可随时切换强度（立即生效）、点 [▸ 查看注入内容] 查看**正在注入主聊天的原文**、或点【完成】/【放弃】
停止引导——主聊天即刻恢复原状。方案存在**当前聊天**的元数据里：刷新页面、切换聊天再回来都不会丢；
切到没有方案的聊天时注入会自动清空，绝不串台。一个聊天同时只有一个方案，采用新方案会替换旧的（有提示）。

**进度。** 方案持续约 20 条主聊天消息后，神谕会提醒一次它还在引导。你可以在参谋模式里问「检查进度」，
神谕会对照最近剧情如实评估：铺垫是否出现、推进到哪了、强度是否合适、是否可以完成。

> 提示：参谋模式每次发送都带完整对话记录，长对话下 token 消耗较大。
> 若你主动选了某个说话人格，它会叠加在参谋指令之上（让こなた或贝蒂给你出方案）——并自动附带职责调整与结构保护：人格只改语气，<StoryPlan> 区块的格式不受影响。
> 参谋模式默认不走补全预设，用它自己的参谋提示词；勾选工具栏里的
> 「套用我的补全预设」可把参谋指令叠加在你整理好的补全预设之上（同世界书模式的用法，仅在需要越狱时使用）。
> 若你的 ST 版本过旧、不支持 `setExtensionPrompt`，采用方案时会明确提示引导无法生效。

## 对话操作（编辑 / 删除 / 重新生成）

三种模式（聊天 / 诊断 / 世界书）及剧情参谋模式通用。把鼠标移到任意一条消息上，气泡右上角会出现小按钮：

- **你的消息**：✎ **编辑**（就地改写，点「保存并重新生成」会更新这句、丢弃其后的所有往来、并据此重新生成；`Esc` 取消、`Ctrl/⌘+Enter` 保存）、🗑 **删除**（连同它的那条回复一起删，保持对话整齐）。
- **神谕的回复**：↻ **重新生成**（丢弃这条回复及其之后的内容，从上一条你的提问重新作答——最新一条的情况就是「重摇上一次」）、🗑 **删除**（只删这一条）。

删除是直接生效、不另弹确认（这本就是与主聊天独立的草稿窗）；想清空整段对话仍用标题栏的清除按钮。生成进行中时这些按钮不响应。
诊断 / 世界书 / 参谋模式下重新生成时，会按当前状态重新读取上下文，应用 / 引导按钮也会随新回复重新挂上。

## 说明

- 系统提示词在每次发送时都会重新构建，所以随着剧情推进，神谕看到的永远是最新的故事状态。
- 历史保存在内存中：关闭/重新打开窗口会保留它；刷新页面或点击 🗑 清空按钮则会重新开始。
- Enter 发送，Shift+Enter 换行。生成过程中，发送按钮会变成停止按钮。
- **移动 / 调整大小**：拖动**标题栏**可移动窗口，拖动**右下角的握把**可调整大小——鼠标和触屏都支持（手机上握把会更大一些）。位置与尺寸都会记住；尺寸自动限制在屏幕范围内。
